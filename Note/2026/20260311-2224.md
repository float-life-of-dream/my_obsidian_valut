# 未命名

数据库本质上是一种以某种方式相关联的数据集合，旨在模拟现实世界的情况

数据库是大多数计算机组建的核心组成部分

一个数据库管理系统是允许存储和修改在数据库中的信息

一个普遍的循序管理系统支持定义，创建、查询，更新，和登录数据库是一直的通过一些数据模型

数据模型是一组包括描述数据在数据库重的组建的集合

	规定：数据库重允许存在那些类型的数据项

模式是特定的数据模型对某个数据库活数据集合的结构描述

	这定义了数据库结构通过数据模型

	另外你会有随机的比特没有任何意义

关系型数据库为主流

NoSQL

层次型

网络型

导航型

结构：数据库的联系和他们的元素独立与物理存在

核心：确定数据库的元素可以被安全存储，不被随意的修改

控制：声明API对进入和修改数据库元素通过关系

将用户和应用从底层数据分离

	用户只需要关心高层次逻辑

数据库管理优化库布局去运行环境，数据库元素和负载

	重新优化会是因素发生改变

关系是一种无序的集合包括了联系属性的关联条目

元祖是关系中单个条目内各属性的取值集合

	值是原子

	特殊值NULL 是每个定义域的成员

	n种关系 ＝ 表中有n列

一个关系的逐渐是识别一个元祖独一无二的表示

一些数据库关系系统会自动创建内部逐渐如果你没有定义

数据库可以自动产生独一无二主键通过设备列

	-  

IDENTITY （SQL 标准）

- SEQUENCE postgreSQL Oracle
- AUTO_INCREMENT MYSQL
外键具体便是属性从一个关系通通过元祖的另一关系
约束节省时间，实在数据类型之外附加约束条件
