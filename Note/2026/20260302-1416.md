# 

### 1. 软件的定义与特征

- **定义：** 软件（software）是在通用计算机硬件之上，面向特定应用目标实现的解决方案。它是计算机系统中与硬件相互依存的另一部分，包括**程序（program）**、**相关数据（data）**及**说明文档（document）**。
    
- **软件开发 vs 硬件设计：** 软件更依赖开发人员的业务素质、智力和团队协作。软件开发成本占产品总成本的大部分，不能简单套用硬件管理模式。
    
- **软件生产 vs 硬件制造：** 硬件制造过程复杂且易引入质量问题；软件产品一旦成型，其制造只是简单的**零成本复制**，仓储和运输极简。
    
- **软件维护 vs 硬件维修：** * 硬件会磨损老化，故障率呈“浴盆曲线”（初期高、中期稳、后期因磨损再升高）。
    
    - 软件是逻辑实体，**不会磨损老化**，但维护比硬件复杂，且维护过程中可能引入新错误。软件产业因此更倾向于**服务业**。

### 3. 工程与软件工程

- **工程的普遍定义：** 是对技术（或社会）实体的分析、设计、建造、验证和管理。它是一种组织良好、管理严密、各方协同的活动。
    
- **18个知识域：** 包括需求、架构、设计、构造、测试、运营、维护、配置管理、工程管理、模型与方法、过程、质量、安全、工程经济等。
    

### 4. 核心方法论

- **抽象 (Abstraction)：** 从众多事物中抽取共同的、本质性的特征，舍弃非本质特征。
    
- **分解 (Decomposition)：** 即“分而治之”，将复杂问题拆解为若干简单问题，逐个击破。
