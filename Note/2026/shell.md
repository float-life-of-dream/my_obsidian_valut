#  Shell

## Shell脚本入门

### 介绍

通过编写Shell命令发送给linux内核执行，操作计算机硬件，所以Shell命令手机用户操作计算机的桥梁

Shell是命令，类似于windows系统的dos命令

Shell是一门程序设计语言，Shell里面含有变量，函数，逻辑控制语句等等

#### Shell脚本

通过Shell命令或程序编程语言编写的Shell脚本文件，这就是shell脚本，也叫shell程序

通过Shell命令提高linux命令的管理效率

#### shell运行过程

当用户下达指令给改操作系统的时候，实际上是把指令告诉shell经过shell解释，处理后让内核做出相应动作。系统的回应和输出信息也有shell处理，然后显示在屏幕上

#### shell解析器

```shell
cat /etc/shells
```

| 解析器类型 | 介绍 |
| ---------- | ---- |
|/bin/sh            |Bourne shell Unix最早使用的shell      |
|/bin/bash|Bourne Again Shell 他是Bourne Shell 的拓展，简称bash，是LinuxOS默认shell，有强灵活性和强大的编辑接口，同时有很友好的界面，交互性很强|
|/sbin/nologin|未登录解析器，shell设置为/sbin/nologin是用于控制用户禁止登陆系统的，有时候有些服务，比如邮件服务，大部分用于接受主机的邮件而已，不需要登陆|
|/bin/dash|dash(Debian Almquist Shell),也是一种Unix shell。它比Bash小，只需要较少的磁盘空间，但是他的对话功能也较少，互动性较差|
|/bin/csh|C Shell 是C语言风格Shell|
|/bin/tcsh|是C Shell 的一个拓展版本|

#### 脚本文件的常用三种执行方式

1. sh解释器执行方式

   语法：`sh 脚本文件`

   介绍：就是利用sh命令执行脚本文件，本质是使用shell解析器运行脚本文件

2. bash解释器执行方式

   语法：`bash 脚本文件`

   介绍：就是利用bash命令执行脚本文件，本质是使用shell解析器运行脚本文

3. 仅路径执行方式

   语法：./脚本文件

   介绍：执行当前目录下的脚本文件

   注意：脚本文件自己执行需要具有可执行权限，否则无法执行

解释器执行不需要执行权限，但路径执行需要执行权限

#### 自定义局部变量

**介绍**

就是定义在一个脚本文件中的变量，只能在这个脚本文件中使用的变量，就是局部变量

**定义**

定义语法

```shell
var_name=value
```

变量定义规则

1. 变量名称可以有字母，数字和下划线组成，但是不能以数字开头
2. 等号两侧不能有空格
3. 在bash环境中，变量的默认类型都是字符串类型无法直接进行数值计算
4. 变量的值如果有空格，必须使用双引号括起来
5. 不能使用shell的关键字作为变量名称

**查询变量值语法**

~~~shell
# 语法1：直接使用变量名查询
$var_name
# 语法2：使用花括号
${var_name}
# 区别花括号方式适合拼接字符串
~~~

**变量删除**

语法：

~~~shell
unset var_name
~~~

#### 自定义变量

就是设置变量值以后不可以修改的变量叫常量，也叫只读变量

语法

~~~shell
readonly var_name
~~~

**父子shell环境**

存在两个shell文件，一个调用另一个时，被调用的那个称为子环境，调用那个那个称为父环境

**介绍**

就是在当前脚本文件中定义全局变量，这个全局变量可以在父子shell环境都可以使用

**用法**


#### 特殊变量：`$*`、`$@`

**语法**

~~~shell
$*
$@
# 含义是获取所有输入参数，用于以后输出所有参数
~~~

`$*`和`$@`

~~~shell
1.不使用双引号括起来，功能一样
  	$*和$@获取所有输入参数，格式为：$1$2……$n
2.使用双引号括起来
	$* 获取所有参数拼接为一个字符串，格式为："$1$2……$n"
	$@ 获取一组参数的列表对象，格式为："$1"""$2"……"$n"
	使用循环打印输出所有参数可以看出去别
~~~

循环语法

```shell
for var in 列表变量
do				# 循环开始
	命令			# 循环体
done			# 循环结束
```

**语法**

>
> 一般来说，返回零代表命令执行成功，非0则执行失败

### shell字符串变量

#### 格式介绍

**字符串三种格式**

1. 单引号方式
2. 双引号方式
3. 不用引号方式

**字符串三种格式区别**

1. 使用单引号''的字符串

   任何字符都会原样输出，在其中使用变量是无效的

2. 由双引号""包围的字符串

   其中包含了变量，那么变量就会解析得到值，而不是原样输出

   字符串中还可以出现双引号的字符串，但需要转义

3. 不被引号包围的字符串

    字符串中不能出现空格，否则后面的字符串会作为其他命令解析

##### 获取字符串长度

**语法**

```shell
${#字符串变量名}
```

#### 拼接

**拼接方式**

1. 无符号拼接
2. 双引号拼接
3. 混合拼接

#### 字符串截取

**语法**

| 格式                      | 功能                                     |
| ------------------------- | ---------------------------------------- |
| ${变量名：start：length} | 从左边start位置开始截取length个字符长度|
|${变量名：start}|从左边start位置开始截取到最后所有字符|
|${变量名：0-steart：length}|从右边截取length个字符|
|${变量名：0-start：}|从右边start位置截取到最后所有字符|

|${变量名：%%\*chars}|从chars右边最后出现char位置开始，截取左边所有字符|

### 索引数组变量

shell支持数字，数组是若干数据的集合，其中每一份数据都称为数据的元素

> 注意Bash Shell 只支持一维数组，不支持多维数组

#### 数组的定义

语法

在shell中，用括号`()`表示数组，数组元素之间使用空格分隔语法为

~~~shell
array_name=(item1 item2 ...) # 方式一
array_name=([索引下标1]=item1 [索引下标2]=item2 ...) #方式2
~~~

> 注意，赋值号`=`两边不能有空格

**数组的获取**

语法

1. 通过下标获取元素值，index从零开始

```shell
${arr[index]}
```

2. 获取值的同时复制给其他变量

```shell
item=${arr[index]}
```

3. 使用`@`或`*`可以获取数组中的所有元素

~~~shell
${arr[@]}
${arr[*]}
~~~

4. 获取数组的长度或个数

```shell
${#arr[@]}
${#arr[*]}
```

5. 获取数组的长度或个数

```shell
${#arr[索引]}
```

#### 数组的拼接

shell数组拼接（数组合并），就是将两个数组连接成一个数组

**语法**

使用`@`和`*`获取数组的所有元素之后进行拼接

```shell
array_name=(${array1[@]} ${array2[@]})
array_name=(${array1[*]} ${array2[*]})
```

数组的删除

**语法**

删除指定元素数据

~~~shell
unset array_name[index]
~~~

删除整个数组

```shell
unset array_name
```

### read

从标准输入中读取数据并赋值给变量。如果没有重定向，默认从终端控制台读取用户输入的数据：如果设置了重定向，就可以从文件中读取数据

**语法**

```shell
read [-options] [var1 var2 ...]
```

> `option`表示选项，`var`表示用于存储数据的变量，可以有一个，也可以有多个
>
> option和var都是可选的，如果没有提供变量名，那么读取的数据将存放到环境变量`REPLY`变量中
>
> $REPLY保存read最后一个读入命令的数据

**option支持的参数**

| 选项 | 说明 |
| ---- | ---- |
|-a array      |把读取的数据赋值给数组array，从下标0开始      |
| -d delimiter|用字符串delimiter指定读取结束的位置，而不是一个换行符（读取到的数据不包括delimiter）|
| -e|在获取用户输入的时候，对功能键进行编码转换，不会直接显式功能键对应的字符|
|-n num|读取num个字符，而不是整行字符|
|-p prompt|显示提示信息，提示内容为prompt|
|-r|原样读取，不把反斜杠字符解释解释为转义字符|
|-s|静默模式，不会在屏幕上显示输入的字符。当输入密码和其他确认信息时是很必要的|
|-t seconds|设置超时时间，单位为秒。如果没有用户没有在指定时间内输入完成，那么read将会返回一个非0的退出状态，表示读取失败|
|-u fd|使用文件描述符fd作为输入源，而不是标准输入，类似于重定向|

正确退出


### declare

用于声明shell变量，声明变量并设置变量的属性，也可以用来显示shell函数，若不加上认何参数，显示全部shell变量与函数

#### 作用

1. 设置变量的属性
2. 查看全部shell变量与函数
3. 实现关连组变量

**设置变量属性语法**

~~~shell
declare [+/-][aArxif][变量名称=设置值]
~~~

> +/- "-"可以指定变量的属性 "+"取消变量所设的属性
>
> a	array，设置为普通索引数组
>
> A	Array，设置为key-value关联数组
>
> r	readonly，将变量设置为可读，也可以使用redonly
>
> x	export ，设置为环境变量，也可以使用export
>
> i 	int，设置为整型变量
>
> f 	funtion ，设置为一个函数变量

**实现key-value关联数组语法**

```shell
declasr -A 关联数组变量名=([字符串key1]=值1 [字符串key2]=值2)
```

获取指定key值

```shell
${关联数组变量名{key}}
```

获取所有值

```shell
${关联数组变量名{@}}
${关联数组变量名{*}}
```

### if条件判断句的退出状态

**介绍**

Linux命令的执行都有一个退出状态，无论是内部命令，还是外部文件命令，还是自定义的shell函数，当他退出时，都会返回一个较小的值给调用他的程序，这是命令的退出状态

大多数命令状态0代表成功，非0代表失败，也有特殊命令，比如diff命令用来比较两个文件的不同，对于"没有差别"的文件返回0，对于"找到差别"的文件返回，对无线文件名返回2

shell中，有多种方式取得命令的退出状态。其中`$?`是最常见的一种

| 运算符    | 使用格式       | 说明                                                         |
| --------- | -------------- | ------------------------------------------------------------ |
| && 或-a   | 条件1&&条件2   | 逻辑于运算符，当条件1和条件2同时成立时，整个表达式才成立。如果检查到条件1的退出状态为0，就不会在检测条件2了，因为不管条件2的退出状态是什么，整个表达式必然不成立，检测也是多此一举 |
| \|\|或 -o | 条件1\|\|条件2 | 逻辑或运算符，条件1和条件2两个表达式只要有一个成立，整个表达式就成立了。如果检测到条件1的退出状态为1，就不会再检测条件2了，因为不管条件2的退出状态是什么，整个表达式必然成立，检测也是多此一举 |
| ！        | !条件          | 逻辑非运算符，相当于取反的效果，如果条件成立，那么整个表达式就不成立，如果条件不成立，那么整个表达式就不成立 |

### case语句

shell case为多选择语句。可以用case语句匹配一个值与一个模式，如果匹配成功，执行相匹配的命令；当分支较多，并且判断条件比较简单时，使用case in语句比较方便

**语法**

```shell 
case 值 in
匹配模式1)
	命令1
	命令2
	...
	;;
匹配模式2)
	命令1
	命令2
	...
	;;
*)
	命令1
	命令2
	...
	;;
esac
```

每一匹配模式必须以右括号结束，取值可以为常量或常数。匹配发现取值符合某一模式后边，其间所有命令开始执行直至;;(类似于break，不可以代替否则语法报错)，取值将检测匹配的每一个模式，一旦模式匹配，则执行完匹配模式相应命令后不在继续其他模式，如无一匹配模式，使用星号*捕获该值，在执行后面的命令。

case、in和esac都是shell关键字esac就是case反写在这里代表结束case

匹配模式：可以是一个数字、字符串甚至是一个简单正则表达式

| 格式  | 说明                             |
| ----- | -------------------------------- |
| *     | 表示任意字符串                   |
| [abc] | 表示a、b、c三个字符的任意一个    |
| [m-n] | 表示从m-nde任意一个字符          |
| \|    | 表示多重选择，类似逻辑中的或运算 |

shell程序与函数的区别

函数与shell程序相比，区别在于：

shell程序（命令）在子shell中运行，会开始独立的进程运行

shell函数在当前的shell进程中进行

## shell重定向输入输出

从键盘读取用过户输入的数据，然后把数据拿到shell程序中使用

**标准输出介绍**

shell程序产生的数据，这些数据拿到shell程序中使用

**默认输入输出文件**

每个Unix和Linux命令运行都会打开三个文件

| 文件名 | 类型                                 | 文件描述符（file description，fd） | 功能                       |
| ------ | ------------------------------------ | ---------------------------------- | -------------------------- |
| stdin  | 标准输入文件<br>(standard input)     | 0                                  | 获取键盘的输入数据         |
| stdout | 标准输出文件<br>(standard output)    | 1                                  | 将正确的数据输出到显示器上 |
| stderr | 标准错误输出文件<br>(standard error) | 2                                  | 将错误信息输出到显示器上   |

> 每个文件都有一个唯一的文件描述符fd，后面会通过唯一文件描述符fd操作对应信息

shell程序操作输入输出会用到者三个文件

1. shell程序默认从stdin中读取输入数据

2. shell程序默认向stdout文件输入正确数据

3. shell程序默认会向stderr文件中输出错误信息

   这三个文件用于临时传输数据使用

#### 重定向输入输出介绍

1. 标准输入时数据默认从键盘流向程序，如果改变了他的方向，数据就会从其他地方流入，这就是输入重定向
2. 标准输出是数据默认从程序流向显示器，如果改变了他的方向，数据就流向其他地方，这就是输出重定向

#### 重定向的作用

输出重定向是指命令的结果不在输出到显示其上，而是输出到其他地方，一般是文件中。这样做最大的好出就是可以把结果保存起来，当我们需要时随时查询

#### 重定向语法

| 命令                | 说明                                                         |
| ------------------- | ------------------------------------------------------------ |
| 命令>file           | 将正确数据重定向输出到file文件中，覆盖方式                   |
| 命令<file           | 将输出重定向从file中读取文件                                 |
| 命令>>file          | 将正确数据重定向输出到file文件中，追加方式                   |
| 命令<file><file2>   | 从file文件中读取数据，输出数据到file2文件中                  |
| 命令fd>file         | 根据指定的文件描述符fd重定向输出到file文件中，覆盖方式       |
| 命令fd>>file        | 根据指定的文件描述符fd重定向输出到file文件中，追加方式       |
| 命令>file fd1>& fd2 | 将fd1和fd2文件符合描述合并输出到文件                         |
| fd1<& fd2           | 将fd1和fd2文件符合描述合并从文件中读取输入                   |
| <<tag               | 读取终端输入数据，将开始标记tag和结束标记tag之间的内容做为输入。<br>标记名tag任意 |

> 在输出重定向中，>代表的是覆盖输出，>>代表的是追加输出
>
> fd是文件描述符
>
> ​	0，通常是标准输入（STDIN）
>
> ​	1 是标准输出（STDOUT）
>
> ​	2是标准错误输出（STDERR）

## shell工具

### cut

cut是一个强大的文本处理工具，它可以将文本按列进行划分的文本处理。cut命令逐行读入文本，然后按列划分字段并进行提取、输出等操作

#### 语法

```shell
cut [options] filename
```

options参数说明

| 选项参数        | 功能                                                         |
| --------------- | ------------------------------------------------------------ |
| -f 提取范围     | 列号，获取第几列                                             |
| -d 自定义分隔符 | 自定义分隔符，默认为制表符                                   |
| -c 提取范围     | 以字符为单位进行分割                                         |
| -b 提取范围     | 以字节为单位进行分割。这些字节位置将忽略多字节字符边界，除非也指定了-n标志 |
| -n              | 与-b选项连用，不分割多字节字符                               |

提取范围说明

| 提取范围  | 说明                                                       |
| --------- | ---------------------------------------------------------- |
| n-        | 提取指定第n列或字节后面所有数据                            |
| n-m       | 提取指定第n列或字符或字节到第m列或字符或字节中间的所有数据 |
| -m        | 提取指定第m列或字符或字节前面所有数据                      |
| n1，n2... | 提前指定枚举列的所有数据                                   |

#### 实例

切割提取指定单词数据

```shell
cat 文件名 | grep 提取词 | cut -d "" -f 2
```

切割提取bash进程的PID号

```shell
ps -aux | grep 'bash' | head -n 1 | cut -d " " -f 8
```

切割提取IP地址

```shell
ifconfig | grep broadcast | cut -d "" -f 10
```

### sed

sed(stream editor，流编辑器)是Linux下一款功能强大的非交互式文本编辑器（vim是交互式编辑器），可以方便的对文本文件的每一行数据匹配查询之后进行增删改查的操作，支持按行、按字段】按正则表达式匹配文本内容，灵活方便，适合大文件的编辑

sed是一种流编辑器，他一次处理一行内容，将这行放入缓存（存空间称为：模式空间），然后才对这行进行处理，处理完后，将缓存区的内容发送终端

#### sed处理数据原理

![image-20250328194135382](images/image-20250328194135382.png)

#### 语法

```shell
sed [选项参数] [模式匹配/sed程序命令] [文件名]

# 模式匹配，sed会读取每一行的数据到模式空间中，之后判断当前行是否符合模式匹配要求，符合要求就会执行色的程序命令，否则不会执行sed程序命令，如果不写匹配模式，那么每一行都会执行sed程序命令
```

#### 选项参数说明

| 选项参数  | 功能                                                         |
| --------- | ------------------------------------------------------------ |
| -e        | 直接在指令列模式上进行sed的动作编辑。他告诉sed将下一个参数解释为一个sed指令，只有当命令行上给出多个sed指令时才会使用-e选项;一行命令语句可以执行多条sed指令 |
| -i        | 直接对内容修改，不加-i时默认只是预览，不会对文件做实际修改   |
| -f        | 后跟保存了sed指令的文件                                      |
| -n        | 取消默认输出，sed会默认输出所有文本内容，使用-n参数后只显示处理过的行 |
| -rregular | 使用扩展正则表达式，默认情况虚啊sed只识别基本正则表达式*     |

#### sed程序命令功能描述

| 命令 | 功能描述                                      |
| ---- | --------------------------------------------- |
| a    | add新增，a的后面可以接字符串，在下一行出现    |
| c    | change更改，更改匹配行的内容                  |
| d    | delete删除，删除匹配内容                      |
| i    | insert插入，向匹配行前插入内容                |
| p    | print打印，打印出匹配的内容，通常与-n选项和用 |
| s    | subsitute替换，替换掉匹配内容                 |
| =    | 用来打印被匹配行的行号                        |
| n    | 读取下一行，遇到n时会自动跳入下一行           |

#### 特殊符号

#### sed缓存区数据交换

模式空间与缓存空间介绍

1. sed处理文件是逐行处理的，即读取一行处理一行，输出一行
2. sed把文件读出来每一行存放的空间叫模式空间，会在该空间中对读到的内容进行处理
3. sed还有一个空间即暂存空间，赞村空间里面只有一个空行
4. sef可使用相应的命令从模式空间王暂存空间放入内容或从暂存空间去内容放入模式空间

关于缓存区sed命令的介绍

| 命令 | 含义                                                 |
| ---- | ---------------------------------------------------- |
| h    | 将模式空间里面的内容复制到暂存空间缓存区（覆盖方式） |
| H    | 将模式空间里面的内容复制到暂存空间缓存区（追加方式） |
| g    | 将暂存空间里面的内容复制到模式空间缓存区（覆盖方式） |
| G    | 将暂存空间里面的内容复制到模式空间缓存区（追加方式） |
| x    | 交换2个空间的内容                                    |

### awk

awk是一个强大的文本分析工具，相对于grep的查找，sed编辑，awk在其对数据分析并生成报告时，显得尤为强大简单来说awk就是文件逐行读入，以空格为默认分隔符进行切片，切开的部分在进行各种分析处理，因为切开的部分使用awk可以定义变量，运算符，使用流程控制语句进行深度加工和分析

#### 语法

```shell
awk 'pattern{action}' {filename}
```

> pattern:表示AWK在数据查找中的内容，就是匹配模式
>
> action:在找到匹配内容时所执行的一系列命令

选项参数说明

| 选项参数 | 功能                   |
| -------- | ---------------------- |
| -F       | 指定摄入文件拆分分隔符 |
| -v       | 赋值一个用户自定义变量 |

#### awk内置变量

| FILENAME | awk浏览的文件名                                              |

| FS       | 设置输入域分隔符，等价于命令行-F选项                         |
| NF       | 浏览记录域的个数，根据分割符分割后的列数                     |

grep，sed，awk，cut文本字符串操作四剑客
