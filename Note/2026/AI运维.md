## 运维基础与环境搭建

### IT运维的基本概念

IT运维是指通过专业技术手段，确保企业的IT系统和网络持续、安全、稳定运行，保障业务的连续性。运维涵盖计算机网络、应用系统、硬件环境和服务流程的综合管理

- 系统运维
    聚焦基础设施稳定性，通过监控工具(如Zabbix、Prometheus)实时追踪服务器、网络及操作系统状态，实施负载均衡、冗余设计、快速响应硬件故障或性能瓶颈，确保业务的连续性
- 自动化运维
    以效率提升与标准化为目标，借助Ansible自动化工具部署、配置管理及巡检任务，结合CI/CD流水线实现发布流程无人化，减少人为失误并提升迭代速度
- 数据库运维
    核心时数据安全与性能调优，通过定期备份数据、主从复制保障数据的可靠性，利用慢查询分析、索引优化及资源监控(如MySQL Performance Schema)提升数据库响应效率
- 容器运维
    专注容器编排与弹性调度，基于Kubernetes实现容器的生命周期管理、服务发现及自动扩缩容，配合Istio服务网格治理微服务，确保分布式应用的高可用与快速伸缩
- 云计算运维
    围绕资源弹性与成本优化，利用云平台(如AWS/Aliyun)的自动扩缩容、Serveless服务动态匹配资源需求、通过资源标签、账单分析工具优化云开支，平衡性能与成本
- 信创运维
    解决国产化适配与生态兼容，在鲲鹏CPU、统信OS、达梦数据库等信创体系中，完成软硬兼容性测试，系统迁移，满足政策与安全要求

环境(Environment)：指的是一组相对独立的软件和硬件资源、以及相应的配置、数据。在这个环境里，可以安装，运行和测试应用

| 环境     | 主要用户           | 主要用途                                                     |
| -------- | ------------------ | ------------------------------------------------------------ |
| 开发环境 | 开发用户           | 编写和调试初步代码<br />快速验证想法                         |

优势：降低风险，提高效率，保证质量

### 工单系统

工单系统（Ticketing System）是运维工程师和IT支持团队在日常工作中广泛使用的工具

简单来说，他是一种用于记录、分配和处理工作请求、问题报告、变更申请等事项的系统

1. 提交工单
    用户在系统中填写问题描述、所属类别、优先级等
2. 自动分配
    系统会根据规则将工单自动分配相关人员
3. 处理反馈
    处理人记录诊断与解决方案，并于把用户沟通
4. 验证关闭
    用户确认问题已解决后，工单状态变为"已关闭"

### 网络通信"三要素"

网络通信是指在不同设备之间通过网络连接进行数据传输与交换的过程

#### 协议 

Protocol：协议定义了数据在网络中传输时的数据规则和格式。常见协议有TCP/IP、UDP、HTTP/s、FTP等他们规定了数据如何被分割、传输、接收和重组

![image-20251025173138168](images/image-20251025173138168.png)

#### IP

internet Protocol：IP地址用于唯一标识网络中的每一台设备。它充当设备的"地址"，使得数据能够在网络中准确地找到目标设备

![image-20251025173228298](images/image-20251025173228298.png)

#### 端口

Port：端口用于区分同一台设备上不同的应用程序或服务。在计算机网络中，一个IP可以对应多个服务，每个服务通过不同的端口进行通信。端口号是识别应用程序的方式

| 端口范围    | 名称           | 说明                                                         |
| ----------- | -------------- | ------------------------------------------------------------ |
| 0~1043      | 知名(保留)端口 | 通常被系统或服务使用(例如，HTTP使用端口80，HTTPS使用端口443，FTP使用端口21) |
| 1024~49151  | 注册端口       | 通常用于用户和应用程序之间的通信(如数据库服务等)             |
| 49152~65535 | 动态或私有端口 | 用于临时连接或服务端通信                                     |

### Linux目录结构

Linux的目录结构是一个树形结构，没有盘符的概念，只有一个根目录/，所有文件再它下面

常见的目录结构

| 目录         | 作用                                                         |
| ------------ | ------------------------------------------------------------ |
| /bin         | 二进制命令所在目录(普通命令=>普通用户和超级管理员root)       |
| /home和/root | 家目录，/home表示普通用户的家目录，/root表示超级用户的root目录 |
| /opt         | 额外的应用软件包                                             |
| /sbin        | 和管理系统相关的命令，【超级管理员用】，s=super超级          |
| /tmp         | 临时文件目录，这个目录被当作回收占使用                       |
| /usr         | 用户或系统软件应用程序目录，类似Windows中的Program files     |
| /dev         | /device缩写，设备文件目录，磁盘，光驱=>/dev/sr0              |
| /etc         | 系统配置文件目录，启动程序，几乎所有的软件都会把自己的配置文件安装在/etc中 |

### 系统报警服务配置

云监控中的报警服务规则配置的住院号目的时帮助用户实时监控阿里云资源、线下IDC、其他云厂商产品或自定义监控数据的运行状态，并在监控指标达到预设条件时自动触发报警通知。通过这种方式，用户可以及时获取异常信息并快速采取措施，确保业务的稳重性和可靠性

步骤一：设置报警联系人

步骤二：设置报警联系组

报警联系组：多个报警人合并为一个组，不同类型可以单独设置为组，告警的时候可以直接针对多个用户同时告警

步骤三：设置报警规则后

报警规则：帮助用户监控运行状态，并在监控指标达到预设条件时，自动触发报警通知
